# Memory Index

- [cost_visibility.md](cost_visibility.md) — Factory tracks per-invocation cost (tokens in/out, dollars). Configurable caps at run/packet/per-day scope. Foundational for budget-aware downstream features (recovery retry budgets, multi-spec scheduling, manager-executor tiering).
- [doctor_diagnostic_deferred.md](doctor_diagnostic_deferred.md) — Defer claw-code's unified preflight `doctor` command. Operator UX value is real but not on the architectural critical path. Sister spec post-Phase 8.
- [event_observability.md](event_observability.md) — Factory emits typed events with provenance labels during pipeline execution. Foundational for Phase 6 recovery layer (events become the substrate) and the future memory write-side (extract facts from event streams).
- [factory_self_governance.md](factory_self_governance.md) — Factory does not govern its own development. The pipeline runner (run.ts) is not invoked against this repo; specs are implementation roadmaps for human+Claude execution. The canonical config is templates/factory.config.json; the root factory.config.json is a non-canonical development convenience.
- [memory_scope_split.md](memory_scope_split.md) — Memory in factory operates at two scopes: per-packet workers stay stateless; the project carries learned memory across runs. Project-scope further subdivides: host-project memory (factory writes to host's artifact tree; host owns loading) vs factory-development memory (memdir-style symlink; per-developer setup, NOT inherited by host projects). Categories: architectural facts, recurring failures, code patterns.
- [recovery_recipes_not_dsl.md](recovery_recipes_not_dsl.md) — Phase 6 recovery uses scenario-keyed recipes (TypeScript functions), not claw-code's PolicyEngine-style DSL with And/Or conditions. Simpler, equally data-driven, no DSL parser to maintain.
- [single_entry_pipeline.md](single_entry_pipeline.md) — Factory has one entry point — run.ts — accepting one or more spec IDs. Four-layer architecture (driver/phases/lifecycle/recovery), CLI-as-agent-protocol, sequential dependency-aware first. Recovery: 8 scenarios, two-layer provider failover (cross-CLI + within-CLI for abstraction providers); LintFailed and TestFailed always escalate.
- [spec_artifact_model.md](spec_artifact_model.md) — Specs are new human-authored markdown artifacts at a higher level than intents. Factory translates each spec into exactly one intent (1:1 parity).
- [verification_grading_deferred.md](verification_grading_deferred.md) — Defer claw-code's "Green Contract" graded verification (TargetedTests/Package/Workspace/MergeReady tiers). Factory uses binary build/lint/test pass/fail today. Revisit after Phase 8 with accumulated host-project experience.
